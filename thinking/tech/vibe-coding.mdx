# Vibe Coding：当编程变成心流表达

## 什么是 Vibe Coding

从 2025 年起，写代码的方式发生了显著变化。开发者不再逐行敲击，而是**用自然语言去"引导" AI**，代码的实现变得更像一种心流的表达。这种新范式被称为 **Vibe Coding**。

Andrej Karpathy 最早使用这个词来描述这种编程方式：你不再关注每一行代码的语法细节，而是专注于表达意图，让 AI 来处理实现。

## 从概念到实体：Vibe Coding 键盘

B 站 UP 主 WellsWang 将这个概念推向了极致——他真的做出了一把 **Vibe Coding 专用键盘**。

起因是一张网络流传的概念图：一把只有几个大按键的"未来键盘"，按键上写着 "Approve"、"Reject"、"Explain" 之类的指令。这像是一个玩笑，但也像是对未来的隐喻——当 AI 承担了大部分编码工作，开发者的交互界面是否也该彻底改变？

WellsWang 决定把这个概念变成现实，项目完全开源（GPL-3.0）。

### 硬件实现

这不是一个概念产品，而是一个可以真正构建的硬件项目：

- **主控**：ESP32-S3，原生 USB 接口，支持 USB HID 键盘/鼠标模式
- **输入**：9 个机械键 + 1 个拨动开关 + 1 个旋转编码器（支持点击、顺时针、逆时针三种操作）
- **显示**：ST7789 驱动的 TFT-LCD 屏幕（76×284 像素），实时显示当前键位布局
- **存储**：EEPROM 持久化配置，支持最多 10 套键位布局切换
- **配置**：Python + Tkinter 桌面端配置工具，通过串口读写设备配置

每个物理按键支持最多 5 键组合和 7 字符显示名，可以映射任意 USB HID 标准键码。项目提供了完整的 PCB 设计文件（EDA + Gerber）、3D 打印外壳模型（STL）和默认配置。

## 背后的思考

Vibe Coding 键盘本身是一个有趣的硬件实验，但它折射出的问题更值得关注：

**编程的本质是什么？** 如果代码只是意图的载体，而 AI 能从自然语言直接生成代码，那么"编程"这个行为的核心就从"写代码"转移到了"清晰地表达需求"和"判断结果是否正确"。

**工具会重塑工作流。** 当年从打字机到文字处理器，改变的不只是输入方式，而是写作本身的流程。Vibe Coding 对编程的改变可能同样深远。

**人的角色在变化。** 审查、决策、引导方向——这些变成了核心技能。逐行写代码变成了可选项，而不是必选项。

**硬件跟随软件范式演进。** 从打字机到 QWERTY 键盘到触摸屏，输入设备始终在适应新的交互模式。Vibe Coding 键盘虽然更像一个思想实验的物化，但它提出了一个真实的问题：当 AI 编码成为主流，我们是否需要重新思考开发者的物理交互界面？
